
**Imperatives Programmieren** ist ein Programmierstil, der beschreibt **_wie_** ein Ergebnis erreicht wird. Der Code gibt Schritt für Schritt vor, welche Operationen in welcher Reihenfolge ausgeführt werden sollen – ähnlich wie eine detaillierte Anleitung. Abstraktion ist zwar möglich, spielt aber eine untergeordnete Rolle.

**Deklaratives Programmieren** ist das Gegenstück dazu: Hier steht das **_was_** im Vordergrund – also das gewünschte Ergebnis. Der Weg dorthin wird nicht explizit beschrieben, sondern dem System überlassen. Das ermöglicht ein höheres Maß an Abstraktion, da Implementierungsdetails verborgen bleiben.

Klassisches Beispiel:
- for loop: hier wird manuell ueber eine liste iteriert. dies ist imperativ
- filter() oder eine SQL Abfrage ist deklarativ
